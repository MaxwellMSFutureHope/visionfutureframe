## 3. Design-System (Priorität: MITTEL)

### Frame-Interface Design:
- ⏳ Frame-IDs (FRAME-03/PROOF) auf Seiten
- ⏳ Statuszeile im Header (MODE: PUBLIC | BUILD: v2026.01)
- ⏳ Evidence-Chips direkt im UI

### Design Tokens:
- ⏳ Typografie-Skala (H1/H2/Text) einheitlich definieren
- ⏳ Spacing (8/12/16/24/32/48) einheitlich definieren
- ⏳ Radius (12/16/24) einheitlich definieren
- ⏳ Border (1px, low-contrast) einheitlich definieren
- ⏳ Glow nur minimal

### Bilder:
- ⏳ Hero Visual
- ⏳ 3 Problem-Icons/Illustrationen
- ⏳ 1 Proof/Flow Grafik
- ⏳ WebP/AVIF Format
- ⏳ Lazy Loading
- ⏳ Fixe Bildhöhen (kein Layout Shift)

## 4. Performance (Priorität: MITTEL)

### Lighthouse Targets:
- Performance ≥ 95
- Accessibility ≥ 95
- SEO ≥ 95
- Best Practices ≥ 95

### Umsetzung:
- ⏳ Bilder als WebP/AVIF
- ⏳ Fonts self-hosted, font-display: swap
- ⏳ Keine schweren Animationen
- ⏳ Keine Videos autoplay
- ⏳ Lazy loading für alles unter dem Fold
